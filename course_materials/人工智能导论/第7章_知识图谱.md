# 第7章 知识图谱

## 7.1 知识图谱概述

### 7.1.1 什么是知识图谱

知识图谱（Knowledge Graph）是一种结构化的知识表示方法，以图结构存储实体及其之间的关系。

**知识图谱的特点**：
- 结构化知识表示
- 实体关系建模
- 语义查询
- 推理能力

### 7.1.2 知识图谱的应用

**搜索引擎**：
- 语义搜索
- 知识问答
- 智能推荐

**智能问答**：
- 事实问答
- 推理问答
- 多跳问答

**推荐系统**：
- 基于知识的推荐
- 可解释推荐

**金融风控**：
- 风险评估
- 欺诈检测
- 关联分析

## 7.2 知识表示

### 7.2.1 本体论

**定义**：对领域知识的形式化表示。

**组成**：
- 概念（Concept）
- 关系（Relation）
- 实例（Instance）
- 属性（Attribute）

### 7.2.2 RDF

**资源描述框架（Resource Description Framework）**：
- 三元组格式：(主语, 谓语, 宾语)
- 统一资源标识符（URI）
- 可扩展

**示例**：
(张三, 学习, 人工智能)

### 7.2.3 OWL

**Web本体语言（Web Ontology Language）**：
- 基于RDF
- 更丰富的表达能力
- 推理支持

**特性**：
- 类层次结构
- 属性约束
- 等价关系

## 7.3 知识图谱构建

### 7.3.1 知识抽取

**实体抽取**：
- 命名实体识别（NER）
- 实体链接

**关系抽取**：
- 监督学习方法
- 远程监督
- 深度学习方法

**属性抽取**：
- 实体属性提取
- 知识补全

### 7.3.2 知识融合

**定义**：将不同来源的知识整合到统一的知识图谱中。

**技术**：
- 实体对齐
- 关系融合
- 属性合并

### 7.3.3 知识存储

**图数据库**：
- Neo4j
- JanusGraph
- ArangoDB

**特点**：
- 高效的图查询
- 关系遍历
- 图算法支持

## 7.4 知识图谱查询

### 7.4.1 SPARQL

**定义**：RDF查询语言。

**特点**：
- 类似SQL语法
- 支持模式匹配
- 支持推理

**示例**：
SELECT ?name WHERE {
  ?x :学习 :人工智能 .
  ?x :名字 ?name .
}

### 7.4.2 Cypher

**定义**：Neo4j的图查询语言。

**特点**：
- 图模式匹配
- 路径查询
- 聚合操作

**示例**：
MATCH (p:Person)-[:学习]->(c:Course)
WHERE c.name = '人工智能'
RETURN p.name

### 7.4.3 图算法

**常用算法**：
- 最短路径
- 社区发现
- 中心性分析
- 路径分析

## 7.5 知识推理

### 7.5.1 基于规则的推理

**定义**：使用逻辑规则进行推理。

**规则类型**：
- 演绎推理
- 归纳推理
- 溯因推理

**示例**：
规则：如果A是B的父亲，B是C的父亲，则A是C的祖父。

### 7.5.2 基于嵌入的推理

**定义**：将实体和关系表示为向量，通过向量运算进行推理。

**方法**：
- TransE
- TransR
- DistMult
- ComplEx

**优点**：
- 计算效率高
- 处理大规模数据

### 7.5.3 基于神经网络的推理

**方法**：
- 图神经网络
- 注意力机制
- 预训练模型

**特点**：
- 强大的表达能力
- 端到端学习
