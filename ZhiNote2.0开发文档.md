# ZhiNote 智能学习助手开发文档

## 1. 项目概述

ZhiNote 是一款智能学习助手小程序，旨在帮助用户更高效地管理学习内容、生成学习计划、分析学习数据，并提供 AI 辅助学习功能。

### 1.1 核心功能

- **多格式内容录入**：支持语音录入、音频上传、文本输入、文件导入
- **AI 学习分析**：语音转文字、课堂总结、考点识别、内容压缩、知识点提取
- **学习计划管理**：AI 生成学习计划、任务打卡、进度追踪
- **学习数据统计**：学习时长、笔记数量、复习次数、知识点掌握度分析
- **知识图谱**：知识点关联可视化

### 1.2 技术栈

- **前端**：微信小程序原生开发
- **后端**：Python FastAPI
- **数据库**：MySQL
- **AI 模型**：千问大模型
- **图表库**：ECharts

## 2. 系统架构

### 2.1 前端架构

ZhiNote 小程序采用微信小程序原生开发框架，主要包含以下模块：

- **页面层**：负责用户界面展示和交互
- **逻辑层**：处理业务逻辑和数据处理
- **网络层**：与后端 API 通信
- **工具层**：提供通用功能和工具方法

### 2.2 后端架构

后端采用 FastAPI 框架，主要包含以下模块：

- **路由层**：处理 HTTP 请求和响应
- **服务层**：实现业务逻辑
- **数据层**：处理数据库操作
- **AI 集成**：与千问大模型等 AI 服务交互

### 2.3 数据流程

1. 用户通过前端页面发起请求
2. 前端通过网络层调用后端 API
3. 后端路由层接收请求并调用相应服务
4. 服务层处理业务逻辑，可能调用 AI 服务或数据库操作
5. 后端返回处理结果给前端
6. 前端展示结果给用户

## 5. API 接口设计

### 5.1 学习计划相关接口

| 接口路径 | 方法 | 功能描述 | 请求参数 | 成功响应 |
|---------|------|---------|---------|----------|
| `/api/learning-plan/generate` | POST | 生成学习计划 | `examDate`, `dailyHours`, `pendingTasks`, `examSubject` | `{"code": 0, "data": {"planData": [...], "remainingDays": 30, "totalProgress": 0}}` |
| `/api/learning-plan/get` | GET | 获取学习计划 | 无 | `{"code": 0, "data": {"hasPlan": true, "planData": [...], "remainingDays": 30, "totalProgress": 0}}` |
| `/api/learning-plan/save-progress` | POST | 保存学习计划进度 | `planData`, `totalProgress` | `{"code": 0, "data": {"success": true}}` |
| `/api/learning-plan/toggle-task` | POST | 切换任务完成状态 | `date`, `taskId` | `{"code": 0, "data": {"success": true}}` |

### 5.2 学习统计相关接口

| 接口路径 | 方法 | 功能描述 | 请求参数 | 成功响应 |
|---------|------|---------|---------|----------|
| `/api/study-stats/check-in` | POST | 学习打卡 | `studyDuration`, `taskCount` | `{"code": 0, "data": {"success": true, "message": "打卡成功"}}` |
| `/api/study-stats/stats` | GET | 获取学习统计数据 | `days` (可选) | `{"code": 0, "data": {"totalDuration": 10, "totalReviews": 5, "totalTasks": 20, "dailyStats": [...]}}` |
| `/api/study-stats/summary` | GET | 获取学习统计摘要 | 无 | `{"code": 0, "data": {"totalDuration": 10, "totalReviews": 5, "totalTasks": 20, "studyDays": 5}}` |

### 5.3 笔记相关接口

| 接口路径 | 方法 | 功能描述 | 请求参数 | 成功响应 |
|---------|------|---------|---------|----------|
| `/api/notes` | GET | 获取笔记列表 | 无 | `{"code": 0, "data": {"notes": [...]}}` |
| `/api/notes` | POST | 创建笔记 | `title`, `content`, `tags` | `{"code": 0, "data": {"id": 1, "title": "..."}}` |
| `/api/notes/{id}` | GET | 获取笔记详情 | 无 | `{"code": 0, "data": {"id": 1, "title": "...", "content": "..."}}` |

| `/api/notes/{id}` | DELETE | 删除笔记 | 无 | `{"code": 0, "data": {"success": true}}` |

## 7. 核心功能实现

### 7.1 学习计划生成

**功能描述**：基于用户输入的考试日期、每日可用时间、待完成任务和考试科目，使用千问大模型生成个性化学习计划。

**实现步骤**：
1. 用户在学习计划页面填写相关信息
2. 前端调用 `/api/learning-plan/generate` 接口
3. 后端调用千问大模型生成学习计划
4. 后端保存学习计划到数据库
5. 前端展示生成的学习计划

**关键代码**：
- 前端：`pages/learning-plan/learning-plan.js` 中的 `generatePlan` 方法
- 后端：`app/services/learning_plan_service.py` 中的 `generate_learning_plan_service` 函数

### 7.2 学习打卡

**功能描述**：用户完成每日学习任务后进行打卡，系统记录学习时长和任务完成情况。

**实现步骤**：
1. 用户在学习计划页面完成任务后点击"今日打卡"按钮
2. 前端计算今日完成的任务数和学习时长
3. 前端调用 `/api/study-stats/check-in` 接口
4. 后端更新学习统计表
5. 前端显示打卡成功提示

**关键代码**：
- 前端：`pages/learning-plan/learning-plan.js` 中的 `checkIn` 方法
- 后端：`app/routers/study_stats.py` 中的 `check_in` 函数

### 7.3 AI 课堂总结

**功能描述**：使用 AI 分析学习内容，生成课程大纲、章节摘要、重点整理和难点分析。

**实现步骤**：
1. 用户在 AI 分析页面选择"AI 课堂总结"功能
2. 前端调用 `/api/ai-analysis/class-summary` 接口
3. 后端调用 AI 模型分析内容
4. 后端返回分析结果
5. 前端展示课堂总结内容

**关键代码**：
- 前端：`pages/analysis/analysis.js` 中的 `goClassSummary` 方法
- 后端：`app/services/ai_analysis_service.py` 中的 `generate_class_summary` 函数

### 7.4 学习数据统计

**功能描述**：统计用户的学习时长、笔记数量、复习次数等数据，并生成可视化图表。

**实现步骤**：
1. 用户进入学习报告页面
2. 前端调用 `/api/study-stats/summary` 接口获取统计数据
3. 前端使用 ECharts 生成各种图表
4. 前端展示学习数据和图表

**关键代码**：
- 前端：`pages/report/report.js` 中的数据加载和图表初始化
- 后端：`app/routers/study_stats.py` 中的 `get_summary` 函数
