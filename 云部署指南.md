# ZhiNote 云部署快速指南

## 🚀 部署步骤（预计2小时完成）

### 第一步：准备MySQL数据库（30分钟）

#### 选项A：使用现有本地MySQL（需要内网穿透）
如果您的MySQL在本地电脑，需要使用 ngrok 或 frp 暴露到公网

#### 选项B：使用免费云MySQL（推荐）
1. 访问 https://supabase.com 或 https://neon.tech
2. 注册免费账号
3. 创建新数据库
4. 获取连接信息（host, port, user, password, database）

#### 选项C： Railway MySQL（最简单）
1. 在Railway项目里添加MySQL插件
2. 自动配置环境变量

### 第三步：部署到Railway（20分钟）

1. **注册Railway**
   - 访问 https://railway.app
   - 用GitHub账号登录

2. **创建新项目**
   - 点击 "New Project"
   - 选择 "Deploy from GitHub repo"
   - 选择 `ZhiNote-backend` 仓库

3. **配置环境变量**
   在项目设置中添加以下环境变量：
   ```
   MYSQL_HOST=你的MySQL主机地址
   MYSQL_PORT=3306
   MYSQL_USER=你的数据库用户名
   MYSQL_PASSWORD=你的数据库密码
   MYSQL_DATABASE=zhinote
   DASHSCOPE_API_KEY=sk-1ba4b8933405452fa18df0e90b6ddfbc
   DASHSCOPE_MODEL=qwen-plus
   ```

4. **添加MySQL数据库（如果需要）**
   - 点击 "+ New"
   - 选择 "Database" → "Add MySQL"
   - Railway会自动配置环境变量

5. **等待部署**
   - Railway会自动检测Python项目
   - 自动安装requirements.txt
   - 自动启动服务
   - 约2-5分钟完成

6. **获取API地址**
   - 部署成功后，Railway会给你一个URL
   - 例如：`https://zhinote-backend-production.up.railway.app`

## ⚠️ 常见问题

### 1. Railway部署失败
**原因**：requirements.txt 有问题
**解决**：检查日志，确保所有依赖都能安装

### 2. 数据库连接失败
**原因**：环境变量配置错误
**解决**：检查Railway环境变量是否正确

### 3. 小程序请求失败
**原因**：baseUrl配置错误或网络问题
**解决**：
- 检查app.js中的baseUrl
- 确保Railway服务正常运行
- 在小程序后台配置服务器域名

### 4. 微信开发者工具报错
**错误**：不在以下合法域名列表中
**解决**：
1. 登录小程序管理后台
2. 开发 → 开发管理 → 开发设置
3. 服务器域名 → request合法域名
4. 添加：`https://你的railway地址.up.railway.app`
